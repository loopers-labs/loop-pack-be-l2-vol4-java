# Spec: CPN-8 쿠폰 발급 내역 조회 (admin)

**소스**: `docs/volume-4/01-requirements.md` — CPN-8
**작성일**: 2026-06-10
**상태**: Draft

## 시나리오 요약

관리자가 특정 쿠폰 템플릿의 발급 내역을 페이지 정보와 함께 조회한다. 관리자 인증을 통과한 요청만 허용하며, 대상 템플릿이 활성 상태(삭제되지 않음)로 존재하면 그 템플릿에서 발급된 쿠폰 한 페이지 분량을 발급 시각 내림차순으로, 페이지 메타 정보(현재 페이지 번호·페이지 크기·총 페이지 수·총 항목 수)와 함께 반환한다. 각 항목은 발급 쿠폰 식별자·발급받은 회원 식별자·상태(사용 가능/사용 완료/만료)·발급 시각을 포함한다. 상태 판정 규칙은 내 쿠폰 목록(CPN-7)과 동일하다.

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 템플릿 + 발급분들이 존재, **When** page·size로 발급 내역 요청, **Then** 해당 페이지 항목들과 페이지 메타가 발급 시각 내림차순으로 반환된다(200). 각 항목은 발급 쿠폰 식별자·회원 식별자·상태·발급 시각.
2. **Given** 관리자 인증 + 활성 템플릿 + 발급분 없음, **When** 발급 내역 요청, **Then** 빈 목록과 페이지 메타가 반환된다(200).
3. **Given** 관리자 인증 + 사용 완료·만료된 발급분이 섞여 존재, **When** 발급 내역 요청, **Then** 각 항목 상태가 CPN-7과 동일 규칙(USED > EXPIRED > AVAILABLE)으로 반환된다.

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 발급 내역 요청, **Then** 403 FORBIDDEN.
2. **Given** 대상 템플릿이 존재하지 않거나 이미 삭제됨, **When** 발급 내역 요청, **Then** 404 NOT_FOUND.

### 비즈니스 규칙
- 삭제된 템플릿의 발급 내역은 조회할 수 없다(부재/삭제 404). 템플릿 목록·상세(CPN-4·5)와 동일한 노출 정책 — 발급 쿠폰 자체는 스냅샷으로 독립 생존하며 회원 쿠폰 목록(CPN-7)에서 확인된다.
- 상태 판정 규칙은 내 쿠폰 목록(CPN-7)과 동일하다(`getStatus`, 기준 시각 `DateTimeUtil` 주입).
- 페이지 크기를 지정하지 않으면 한 페이지에 20건을 반환한다.
- 결과는 발급 시각 내림차순으로 정렬한다.

## 엣지 케이스
- 빈 결과(빈 목록 + 메타) / size 미지정(20 적용) / 마지막 페이지 부분 채움.
- 대상 템플릿 부재(404) / 이미 삭제됨(404).
- 상태 혼재(AVAILABLE·USED·EXPIRED) 항목.
- 권한 경계: 회원 헤더만 있고 admin 헤더 없으면 403.

## 기능 요구사항
- **FR-001**: 시스템은 관리자 인증 통과 요청만 허용한다(실패 403).
- **FR-002**: 시스템은 대상 템플릿이 활성 상태로 존재하는지 확인한다(부재/삭제 404).
- **FR-003**: 시스템은 page·size로 페이징하며 기본값은 page=0, size=20이다.
- **FR-004**: 시스템은 해당 템플릿 발급분을 발급 시각 내림차순으로 페이징하고 페이지 메타(현재 페이지·크기·총 페이지·총 항목)를 포함한다.
- **FR-005**: 각 항목에 발급 쿠폰 식별자·회원 식별자·상태·발급 시각을 포함한다.

## 관련 엔티티
- **CouponModel**: 대상 템플릿 활성 존재 확인(`getActiveById` — 부재/삭제 404, CPN-2 재사용).
- **UserCoupon** (기존): 상태 판정 `getStatus(now)` 재사용(CPN-7 도입).
- **UserCouponStatus** (기존 enum): 항목 상태(CPN-7 도입).
- **UserCouponRepository** (메서드 추가): 특정 템플릿(`couponId`) 발급분을 발급 시각 내림차순으로 페이징 조회.
- **재사용**: `AdminAuthInterceptor`(admin 인증), 페이지 응답 구조(`PageResponse` 패턴, CPN-4 선례), `DateTimeUtil`(상태 기준 시각), `ErrorType`(FORBIDDEN·NOT_FOUND).

## 관련 결정
- **삭제 노출 정책 (결정 4 정합)**: 삭제된 템플릿의 발급 내역은 404 — 템플릿 상세(CPN-5)와 동일. 발급 쿠폰은 회원 측(CPN-7)에서만 독립 노출.
- **상태 판정 일관 (결정 2)**: CPN-7과 동일한 `getStatus` 규칙·기준 시각 주입.
- 페이지네이션 표현은 CPN-4 선례를 따른다.

## 성공 기준 / 범위 밖
- **성공**: 위 모든 수용 시나리오·테스트 계획 green. `GET /api-admin/v1/coupons/{couponId}/issues`가 인증·템플릿 존재·페이징·상태 판정을 명세대로 처리.
- **범위 밖**: 회원·상태 필터·검색, 발급 쿠폰 강제 회수, 회원 쿠폰 목록(CPN-7), 주문 적용(ORD-7).
