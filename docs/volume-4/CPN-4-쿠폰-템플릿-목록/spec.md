# Spec: CPN-4 쿠폰 템플릿 목록 (admin)

**소스**: `docs/volume-4/01-requirements.md` — CPN-4
**작성일**: 2026-06-10
**상태**: Draft

## 시나리오 요약

관리자가 페이지 정보를 포함해 등록된 쿠폰 템플릿 목록을 조회한다. 관리자 인증을 통과한 요청만 허용하며, 삭제되지 않은 템플릿 한 페이지 분량을 등록 시각 내림차순으로, 페이지 메타 정보(현재 페이지 번호·페이지 크기·총 페이지 수·총 항목 수)와 함께 반환한다. 각 항목은 식별자·이름·할인 타입·할인 값·최소 주문 금액·만료 시각·등록 시각·갱신 시각을 포함한다. 만료된 템플릿도 포함된다(만료는 발급·사용의 차단 조건이지 존재의 소멸이 아니다).

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 템플릿들이 존재, **When** page·size로 목록 요청, **Then** 해당 페이지 항목들과 페이지 메타가 등록 시각 내림차순으로 반환된다(200). 각 항목은 식별자·이름·할인 타입·할인 값·최소 주문 금액·만료 시각·등록 시각·갱신 시각.
2. **Given** 관리자 인증 + 활성 템플릿 없음, **When** 목록 요청, **Then** 빈 목록과 페이지 메타가 반환된다(200).
3. **Given** 관리자 인증 + 만료 시각이 지난 활성 템플릿 존재, **When** 목록 요청, **Then** 만료된 템플릿도 목록에 포함된다.

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 목록 요청, **Then** 403 FORBIDDEN.

### 비즈니스 규칙
- 삭제된 템플릿은 응답에서 제외한다.
- 만료 시각이 지난 템플릿은 포함한다(존재≠소멸, 관리자 목록은 운영 이력 확인 목적).
- 페이지 크기 미지정 시 20건.
- 결과는 등록 시각 내림차순 정렬.
- 최소 주문 금액을 지정하지 않은 항목은 0(제약 없음)으로 반환한다(항상 포함).

## 엣지 케이스
- 빈 결과(빈 목록 + 메타) / size 미지정(20 적용) / 마지막 페이지 부분 채움.
- 만료된 템플릿 포함 / 삭제된 템플릿 제외.
- 최소 주문 금액 0(미지정) 항목.

## 기능 요구사항
- **FR-001**: 시스템은 관리자 인증 통과 요청만 허용한다(실패 403).
- **FR-002**: 시스템은 page·size로 페이징하며 기본값은 page=0, size=20이다.
- **FR-003**: 시스템은 삭제되지 않은 템플릿만(만료 포함) 등록 시각 내림차순으로 페이징한다.
- **FR-004**: 시스템은 페이지 메타(현재 페이지·크기·총 페이지·총 항목)를 포함한다.
- **FR-005**: 각 항목에 식별자·이름·할인 타입·할인 값·최소 주문 금액(0=제약 없음)·만료 시각·등록 시각·갱신 시각을 포함한다.

## 관련 엔티티
- **CouponModel**: 읽기.
- **CouponRepository**: 활성 템플릿 페이지 조회(`findActiveByPage` — 삭제 제외·등록 시각 내림차순·페이징) 신규.
- **CouponAdminInfo** (신규): 템플릿 상세 출력 DTO — 식별자·이름·할인 타입·할인 값·최소 주문 금액·만료 시각·등록/갱신 시각. CPN-5와 공유.
- **재사용**: `AdminAuthInterceptor`, 페이지 응답 구조(`PageResponse` 패턴, BRD-2 선례).

| Integration | CouponRepository | 활성만(삭제 제외)·만료 포함·등록 시각 내림차순·페이징(크기/오프셋)·총 개수 |

## 관련 결정
- **soft delete**: 삭제 템플릿 제외.
- **만료 포함**: 만료는 발급·사용 차단 조건일 뿐 존재는 유지(결정 2와 정합).
- **CPN-1 review 결정 계승**: 응답의 `discountType`은 `DiscountType` enum(직렬화 시 문자열), `discountValue`는 int.
- 페이지네이션 표현 방식은 plan에서 BRD-2 선례를 따른다.

## 성공 기준 / 범위 밖
- **성공**: 위 모든 수용 시나리오·테스트 green. `GET /api-admin/v1/coupons`가 인증·필터(삭제 제외·만료 포함)·정렬·페이징·메타를 명세대로 처리.
- **범위 밖**: 정렬 기준 다양화, 검색·상태 필터, 발급 수 집계.
