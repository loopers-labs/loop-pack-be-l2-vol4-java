# Spec: CPN-5 쿠폰 템플릿 상세 (admin)

**소스**: `docs/volume-4/01-requirements.md` — CPN-5
**작성일**: 2026-06-10
**상태**: Draft

## 시나리오 요약

관리자가 특정 쿠폰 템플릿의 상세 정보를 조회한다. 관리자 인증을 통과한 요청만 허용하며, 대상 템플릿이 활성 상태(삭제되지 않음)로 존재하면 식별자·이름·할인 타입·할인 값·최소 주문 금액·만료 시각·등록 시각·갱신 시각을 반환한다. 만료 시각이 지난 템플릿도 정상 조회된다(CPN-4와 동일 노출 정책).

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 템플릿 존재, **When** 식별자로 상세 요청, **Then** 식별자·이름·할인 타입·할인 값·최소 주문 금액·만료 시각·등록 시각·갱신 시각이 반환된다(200).
2. **Given** 관리자 인증 + 만료 시각이 지난 활성 템플릿 존재, **When** 상세 요청, **Then** 정상 조회된다.

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 상세 요청, **Then** 403 FORBIDDEN.
2. **Given** 대상 템플릿이 부재거나 이미 삭제됨, **When** 상세 요청, **Then** 404 NOT_FOUND.

### 비즈니스 규칙
- 활성 템플릿만 노출한다(삭제된 템플릿은 404).
- 만료 시각이 지난 템플릿도 정상 조회된다(CPN-4와 동일 정책).
- 최소 주문 금액을 지정하지 않은 템플릿은 0(제약 없음)으로 반환한다.

## 엣지 케이스
- 삭제된 템플릿(404) / 존재하지 않는 id(404).
- 만료된 템플릿(정상 조회) / 최소 주문 금액 0(미지정) 템플릿(0 반환).
- 권한 경계: 회원 헤더만 있고 admin 헤더 없으면 403.

## 기능 요구사항
- **FR-001**: 시스템은 관리자 인증 통과 요청만 허용한다(실패 403).
- **FR-002**: 시스템은 활성 템플릿만 노출한다(부재/삭제 404).
- **FR-003**: 시스템은 식별자·이름·할인 타입·할인 값·최소 주문 금액(0=제약 없음)·만료 시각·등록 시각·갱신 시각을 반환한다.

## 관련 엔티티
- **CouponModel**: 읽기.
- **CouponRepository**: 활성 단건 조회(`getActiveById`) — CPN-2와 공유(부재/삭제 시 NOT_FOUND).
- **CouponAdminInfo** (재사용): CPN-4에서 도입한 템플릿 상세 출력 DTO.
- **표현 계층**: 기존 `CouponAdminV1Controller`에 GET 상세 추가, 상세 응답 DTO 신규.

## 테스트 계획
| 레벨 | 대상 | 무엇을 단언하는가 |
|------|------|------------------|
| Service/Facade 단위 | 상세 조회 유스케이스 | 활성 존재 시 정보 반환(시각·0 최소금액 포함), 부재/삭제 NOT_FOUND |
| E2E | `GET /api-admin/v1/coupons/{couponId}` | 200 + 응답 키(id·name·discountType·discountValue·minOrderAmount·expiredAt·시각) / 만료 템플릿 200 / 403 / 404 (statusCode + meta.result + errorCode까지) |

(활성 단건 조회 Integration은 CPN-2와 공유 — 중복 작성하지 않음)

## 관련 결정
- **만료 포함**: 만료된 템플릿도 정상 조회(결정 2·CPN-4와 정합).
- **soft delete**: 삭제 템플릿 404.
- **CPN-1 review 결정 계승**: 응답의 `discountType`은 enum(문자열 직렬화), `discountValue`는 int.

## 성공 기준 / 범위 밖
- **성공**: 위 모든 수용 시나리오·테스트 green. `GET /api-admin/v1/coupons/{couponId}`가 인증·존재·필드(시각·null 최소금액 포함)를 명세대로 처리.
- **범위 밖**: 발급 내역 포함(CPN-8), 운영 메모 등 추가 관리자 필드.
