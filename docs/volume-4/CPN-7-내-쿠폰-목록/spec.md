# Spec: CPN-7 내 쿠폰 목록 (상태 포함)

**소스**: `docs/volume-4/01-requirements.md` — CPN-7
**작성일**: 2026-06-10
**상태**: Draft

## 시나리오 요약

로그인한 회원이 자신이 발급받은 쿠폰 전체를 상태와 함께 조회한다. 회원 인증을 통과한 요청만 허용하며, 인증된 회원이 발급받은 발급 쿠폰 전부를 발급 시각 내림차순으로 반환한다. 각 항목은 발급 쿠폰 식별자·쿠폰 이름·할인 타입·할인 값·최소 주문 금액·만료 시각·상태(사용 가능/사용 완료/만료)를 포함한다. 상태는 저장 컬럼이 아니라 사용 시각·만료 시각에서 파생되는 표시 값으로, `UserCoupon`이 `getStatus`로 판정한다(결정 2). 1인 1매 정책상 회원당 쿠폰 수가 템플릿 총수를 넘지 않아 페이지네이션 없이 전체를 반환한다.

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 회원 인증 + 본인 발급 쿠폰들이 존재, **When** 목록 요청, **Then** 본인 발급 쿠폰 전체가 발급 시각 내림차순으로 반환된다(200). 각 항목은 발급 쿠폰 식별자·쿠폰 이름·할인 타입·할인 값·최소 주문 금액·만료 시각·상태.
2. **Given** 회원 인증 + 미사용·미만료 발급 쿠폰, **When** 목록 요청, **Then** 해당 항목 상태가 `AVAILABLE`로 반환된다.
3. **Given** 회원 인증 + 사용 완료된 발급 쿠폰(만료 시각 경과 여부 무관), **When** 목록 요청, **Then** 해당 항목 상태가 `USED`로 반환된다(사용 완료가 만료보다 우선).
4. **Given** 회원 인증 + 미사용이지만 만료 시각이 지난 발급 쿠폰, **When** 목록 요청, **Then** 해당 항목 상태가 `EXPIRED`로 반환된다.
5. **Given** 회원 인증 + 발급 이력 없음, **When** 목록 요청, **Then** 빈 목록이 반환된다(200).
6. **Given** 발급받은 뒤 그 템플릿이 삭제된 발급 쿠폰, **When** 목록 요청, **Then** 해당 발급 쿠폰도 목록에 포함된다(스냅샷 독립, 결정 4).

### Exception Flow
1. **Given** 회원 인증 정보가 없거나 올바르지 않을 때, **When** 목록 요청, **Then** 인증 실패로 응답한다(401 UNAUTHORIZED).

### 비즈니스 규칙
- 상태는 세 가지다: 사용 가능(`AVAILABLE`)·사용 완료(`USED`)·만료(`EXPIRED`).
- 사용 완료가 만료보다 우선한다. 사용된 쿠폰은 만료 시각이 지나도 `USED`로 표시된다.
- 만료는 발급 쿠폰에 기록된 만료 시각이 현재 시각을 지났는지로 판정한다(결정 2). 판정 기준 시각은 표현 계층이 `DateTimeUtil`로 요청 단위 확정해 주입한다.
- 템플릿이 삭제된 발급 쿠폰도 목록에 포함된다(결정 4 — 발급 쿠폰은 스냅샷으로 독립).
- 페이지네이션 없이 전체를 반환한다(1인 1매로 회원당 쿠폰 수가 제한적).
- 결과는 발급 시각 내림차순으로 정렬한다.

## 엣지 케이스
- 상태 경계: 미사용·미만료(AVAILABLE) / 미사용·만료(EXPIRED) / 사용·미만료(USED) / 사용·만료(USED 우선).
- 빈 결과(빈 목록) / 다건 정렬(발급 시각 내림차순) / 삭제된 템플릿 발급분 포함.
- 권한 경계: 회원 인증 헤더(`X-Loopers-LoginId/Pw`) 없거나 비밀번호 불일치면 401.

## 기능 요구사항
- **FR-001**: 시스템은 회원 인증 통과 요청만 허용한다(실패 401).
- **FR-002**: 시스템은 인증된 회원이 발급받은 발급 쿠폰 전체를 발급 시각 내림차순으로 반환한다.
- **FR-003**: 시스템은 각 발급 쿠폰의 상태를 사용 시각·만료 시각에서 파생해 판정한다(USED > EXPIRED > AVAILABLE 우선순위).
- **FR-004**: 각 항목에 발급 쿠폰 식별자·쿠폰 이름·할인 타입·할인 값·최소 주문 금액·만료 시각·상태를 포함한다.
- **FR-005**: 삭제된 템플릿의 발급 쿠폰도 포함하고, 발급 이력이 없으면 빈 목록을 반환한다.

## 관련 엔티티
- **UserCoupon** (기존, 메서드 추가): 상태 판정 `getStatus(now)` 신규 — `usedAt`이 있으면 `USED`, 없고 만료 시각 경과면 `EXPIRED`, 아니면 `AVAILABLE`. CPN-6에서 범위 밖으로 미뤘던 메서드를 이번에 추가한다.
- **UserCouponStatus** (신규 enum): `AVAILABLE`·`USED`·`EXPIRED`. 저장 컬럼이 아닌 `getStatus`의 반환 타입(파생 표시 상태).
- **UserCouponRepository** (메서드 추가): 회원 발급 쿠폰 전체를 발급 시각 내림차순으로 조회(삭제 템플릿 발급분 포함).
- **재사용**: `@LoginUser AuthenticatedUser`(회원 인증), `DateTimeUtil`(기준 시각), `ErrorType`(UNAUTHENTICATED).

## 관련 결정
- **결정 2 (만료 판정)**: 발급 쿠폰의 만료 시각이 현재 시각을 지났는지로 EXPIRED 판정. 기준 시각은 `DateTimeUtil` 주입.
- **결정 4 (삭제 독립)**: 템플릿이 삭제돼도 발급 쿠폰은 스냅샷으로 독립 — 목록에 포함.
- **상태 우선순위**: 사용 완료 > 만료 > 사용 가능. `getStatus`가 `usedAt` 우선 판정.
- **페이지네이션 없음**: 1인 1매 정책상 회원당 쿠폰 수가 제한적이라 전체 반환(CPN-4/8의 페이징과 의도된 차이).

## 성공 기준 / 범위 밖
- **성공**: 위 모든 수용 시나리오·테스트 계획 green. `GET /api/v1/users/me/coupons`가 인증·전체 조회·상태 판정·정렬을 명세대로 처리.
- **범위 밖**: 상태/타입 필터·검색, 페이지네이션, 쿠폰 사용(ORD-7)·발급(CPN-6), 관리자 발급 내역(CPN-8).
