## 1. 상품 탐색 — 목록 조회 → 상세 조회

사용자가 상품 목록을 탐색하고, 특정 상품의 상세 정보를 확인하는 흐름입니다.  
브랜드 필터와 정렬 조건을 조합할 수 있으며, 상세 조회 시 브랜드 정보도 함께 제공됩니다.

```mermaid
sequenceDiagram
    actor 사용자
    participant 상품
    participant 브랜드

    사용자 ->> 상품: 상품 목록 요청<br/>(brandId?, sort, page, size)

    alt 브랜드 필터 존재
        상품 ->> 브랜드: 브랜드 존재 확인
        브랜드 -->> 상품: 브랜드 존재함
    end

    상품 -->> 사용자: 상품 목록 반환<br/>(상품명, 가격, 브랜드명, 좋아요 수)

    사용자 ->> 상품: 상품 상세 요청 (productId)

    alt 상품이 존재하지 않거나 삭제됨
        상품 -->> 사용자: 404 Not Found
    else 상품 존재
        상품 ->> 브랜드: 브랜드 정보 조회
        브랜드 -->> 상품: 브랜드 정보 반환
        상품 -->> 사용자: 상품 상세 반환<br/>(상품명, 가격, 재고, 브랜드 정보)
    end
```

## 2. 상품 좋아요 등록 / 취소 (멱등)

로그인한 사용자가 상품에 좋아요를 등록하거나 취소하는 흐름입니다.  
**멱등성**이 보장되어, 이미 등록/취소된 상태에서 동일 요청이 들어와도 에러 없이 정상 처리됩니다.

```mermaid
sequenceDiagram
    actor 사용자
    participant 좋아요
    participant 상품

    %% 좋아요 등록
    사용자 ->> 좋아요: 좋아요 등록 요청 (productId)

    좋아요 ->> 상품: 상품 존재 확인

    alt 상품이 존재하지 않거나 삭제됨
        좋아요 -->> 사용자: 404 Not Found
    else 상품 존재
        좋아요 ->> 좋아요: 이미 좋아요 여부 확인

        alt 이미 좋아요 등록된 상태
            좋아요 -->> 사용자: 200 OK (멱등 — 중복 무시)
        else 신규 좋아요
            좋아요 ->> 좋아요: 좋아요 저장
            좋아요 -->> 사용자: 200 OK
        end
    end

    Note over 사용자, 상품: ── 좋아요 취소 ──

    사용자 ->> 좋아요: 좋아요 취소 요청 (productId)

    좋아요 ->> 좋아요: 좋아요 여부 확인

    alt 이미 취소된 상태 (좋아요 없음)
        좋아요 -->> 사용자: 200 OK (멱등 — 이미 없음)
    else 좋아요 존재
        좋아요 ->> 좋아요: 좋아요 삭제
        좋아요 -->> 사용자: 200 OK
    end
```

## 3. 주문 생성 — 정상 흐름

사용자가 여러 상품을 한 번에 주문하는 흐름입니다.  
모든 상품의 재고가 충분할 때, 재고를 차감하고 주문을 생성합니다.  
**주문 시점의 상품 정보(명칭, 가격)는 스냅샷으로 주문 항목에 저장**됩니다.

```mermaid
sequenceDiagram
    actor 사용자
    participant 주문
    participant 상품
    participant 재고

    사용자 ->> 주문: 주문 요청<br/>([{productId, quantity}, ...])

    주문 ->> 주문: 요청 유효성 검사<br/>(items 1개 이상, quantity 1 이상)

    loop 요청된 상품 각각에 대해
        주문 ->> 상품: 상품 정보 조회 (productId)

        alt 상품이 존재하지 않거나 삭제됨
            상품 -->> 주문: 404 Not Found
            주문 -->> 사용자: 404 Not Found (주문 전체 실패)
        else 상품 존재
            상품 -->> 주문: 상품 정보 반환<br/>(상품명, 가격, 브랜드명)
            주문 ->> 재고: 재고 확인 (productId, quantity)
            재고 -->> 주문: 재고 충분 확인
        end
    end

    Note over 주문, 재고: 모든 상품 재고 확인 완료

    loop 요청된 상품 각각에 대해
        주문 ->> 재고: 재고 차감 (productId, quantity)
        재고 -->> 주문: 차감 완료
    end

    주문 ->> 주문: 주문 생성<br/>+ 주문 항목별 스냅샷 저장<br/>(주문 시 상품명, 가격, 수량)

    주문 -->> 사용자: 201 Created (orderId)
```

## 4. 주문 생성 — 재고 부족 실패 흐름

재고 확인 중 하나라도 수량이 부족하면 **주문 전체가 실패**합니다.  
재고 차감은 일어나지 않으며, 사용자에게 부족한 상품 정보를 알려줍니다.

```mermaid
sequenceDiagram
    actor 사용자
    participant 주문
    participant 상품
    participant 재고

    사용자 ->> 주문: 주문 요청<br/>([{productId, quantity}, ...])

    주문 ->> 상품: 상품 정보 조회 (productId: 1)
    상품 -->> 주문: 상품 정보 반환
    주문 ->> 재고: 재고 확인 (productId: 1, quantity: 2)
    재고 -->> 주문: 재고 충분 (재고: 5)

    주문 ->> 상품: 상품 정보 조회 (productId: 3)
    상품 -->> 주문: 상품 정보 반환
    주문 ->> 재고: 재고 확인 (productId: 3, quantity: 3)
    재고 -->> 주문: ❌ 재고 부족 (재고: 1)

    Note over 주문: 재고 부족 감지<br/>→ 재고 차감 없이 즉시 중단

    주문 -->> 사용자: 400 Bad Request<br/>(재고 부족 상품 안내)
```

## 플로우 간 관계 요약

```
[상품 탐색]
  사용자가 상품을 둘러보고, 원하는 상품을 발견한다.
       ↓
[좋아요 등록]
  마음에 드는 상품에 좋아요를 누른다. (멱등)
       ↓
[주문 생성 — 정상]
  여러 상품을 담아 주문한다. 재고 확인 → 차감 → 스냅샷 저장.
       ↓ (재고 부족 시 분기)
[주문 생성 — 실패]
  재고 부족 상품이 있으면 주문 전체가 거부된다.
```
