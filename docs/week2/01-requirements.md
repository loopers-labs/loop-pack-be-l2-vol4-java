# 요구사항 명세

## 유비쿼터스 언어

| 한글 | 영문 | 설명 |
|---|---|---|
| 브랜드 | Brand | 상품을 판매하는 브랜드 |
| 상품 | Product | 브랜드가 등록한 판매 상품 |
| 재고 | Stock | 상품의 구매 가능 수량 |
| 좋아요 | Like | 사용자가 상품에 표시하는 관심 |
| 주문 | Order | 사용자가 상품을 구매하는 행위 |
| 주문 항목 | OrderItem | 주문 내 개별 상품 + 수량 + 당시 가격 스냅샷 |
| 재고 차감 | StockDeduct | 주문 시 상품 재고를 줄이는 동작 |
| 논리 삭제 | Soft Delete | `deleted_at` 컬럼으로 삭제 처리, 물리 데이터는 보존 |

---

## 설계 결정 사항

| 항목 | 결정 | 이유 |
|---|---|---|
| 주문 트랜잭션 | 단일 트랜잭션 (재고 차감 + 주문 저장) | 구현 단순, 정합성 보장 |
| 브랜드/상품 삭제 | soft delete (`deleted_at`) | 주문 내역 데이터 정합성 보호 |
| 주문 내역 상품 정보 | `order_items` 스냅샷 저장 | 상품 변경/삭제 후에도 당시 정보 보존 |
| 좋아요 수 집계 | `likes` 테이블 COUNT 집계 | 캐시 컬럼 동기화 복잡도 제거, 성능 이슈 시 전환 검토 |
| 재고 노출 | 사용자: 유무만 / 어드민: 수량 | 재고 관리 전략 노출 방지 |

## 도메인별 유저 시나리오

### 브랜드 (Brand)

**사용자 관점**
- 특정 브랜드의 정보를 조회할 수 있다.

**어드민 관점**
- 브랜드를 등록, 수정, 삭제할 수 있다.
- 브랜드 삭제 시 해당 브랜드의 상품도 soft delete 처리된다.
- 등록된 브랜드 목록을 페이징으로 조회할 수 있다.

---

### 상품 (Product)

**사용자 관점**
- 상품 목록을 조회할 수 있다. (브랜드 필터, 정렬, 페이징)
  - 정렬 기준: `latest` (기본), `price_asc`, `likes_desc`
  - 기본 페이지 크기: 20
- 특정 상품의 상세 정보를 조회할 수 있다.
- 재고 유무(있음/없음)만 노출된다. 수량은 노출하지 않는다.

**어드민 관점**
- 상품을 등록, 수정, 삭제할 수 있다.
- 상품 등록 시 브랜드는 이미 등록된 브랜드여야 한다.
- 상품 수정 시 브랜드는 변경할 수 없다.
- 재고 수량이 노출된다.

---

### 좋아요 (Like)

**사용자 관점**
- 로그인한 사용자는 상품에 좋아요를 등록할 수 있다.
- 좋아요 취소는 별도 엔드포인트(DELETE)로 분리된다.
- 내가 좋아요한 상품 목록을 조회할 수 있다.

**기능 흐름**
1. 좋아요 등록: 로그인 확인 → 상품 존재 확인 → 중복 여부 확인 → 저장
2. 좋아요 취소: 로그인 확인 → 좋아요 존재 확인 → 삭제
3. 좋아요 목록: 로그인 확인 → 해당 유저의 좋아요 상품 목록 반환

**예외 조건**
- 존재하지 않는 상품(삭제된 상품 포함)에 좋아요 불가
- 이미 좋아요한 상품 중복 등록 시 409 Conflict
- 좋아요하지 않은 상품 취소 요청 시 404 Not Found

---

### 주문 (Order)

**사용자 관점**
- 로그인한 사용자는 여러 상품을 한 번에 주문할 수 있다.
- 주문 목록을 날짜 범위로 조회할 수 있다.
- 단일 주문의 상세 정보를 조회할 수 있다.

**기능 흐름**
1. 주문 요청 수신
2. 각 상품 존재 여부 확인 (soft delete된 상품 제외)
3. 각 상품 재고 확인 (부족 시 전체 주문 실패)
4. 재고 차감 + 주문 생성 (단일 트랜잭션)
5. `order_items`에 당시 상품명, 가격 스냅샷 저장

**예외 조건**
- 재고 부족 시 주문 전체 실패 (부분 성공 없음)
- 존재하지 않는 상품 포함 시 주문 실패
- 수량은 1 이상이어야 함

**어드민 관점**
- 전체 주문 목록을 페이징으로 조회할 수 있다.
- 단일 주문 상세를 조회할 수 있다.

## API 목록

### 사용자 API (`/api/v1`)

| Method | URI | 인증 | 설명 |
|---|---|---|---|
| GET | `/api/v1/brands/{brandId}` | X | 브랜드 정보 조회 |
| GET | `/api/v1/products` | X | 상품 목록 조회 |
| GET | `/api/v1/products/{productId}` | X | 상품 상세 조회 |
| POST | `/api/v1/products/{productId}/likes` | O | 좋아요 등록 |
| DELETE | `/api/v1/products/{productId}/likes` | O | 좋아요 취소 |
| GET | `/api/v1/users/{userId}/likes` | O | 내가 좋아요한 상품 목록 |
| POST | `/api/v1/orders` | O | 주문 생성 |
| GET | `/api/v1/orders` | O | 주문 목록 조회 (날짜 필터) |
| GET | `/api/v1/orders/{orderId}` | O | 주문 상세 조회 |

### 어드민 API (`/api-admin/v1`)

| Method | URI | 설명 |
|---|---|---|
| GET | `/api-admin/v1/brands` | 브랜드 목록 조회 |
| GET | `/api-admin/v1/brands/{brandId}` | 브랜드 상세 조회 |
| POST | `/api-admin/v1/brands` | 브랜드 등록 |
| PUT | `/api-admin/v1/brands/{brandId}` | 브랜드 수정 |
| DELETE | `/api-admin/v1/brands/{brandId}` | 브랜드 삭제 (상품 soft delete 포함) |
| GET | `/api-admin/v1/products` | 상품 목록 조회 |
| GET | `/api-admin/v1/products/{productId}` | 상품 상세 조회 |
| POST | `/api-admin/v1/products` | 상품 등록 |
| PUT | `/api-admin/v1/products/{productId}` | 상품 수정 |
| DELETE | `/api-admin/v1/products/{productId}` | 상품 삭제 |
| GET | `/api-admin/v1/orders` | 주문 목록 조회 |
| GET | `/api-admin/v1/orders/{orderId}` | 주문 상세 조회 |
