# PRD — 감성 이커머스 Vol.2 (Brand / Product / Like / Order)

> Tag: BE_L2 scenario
> 작성일: 2026-05-20

---

## 1. 제품 개요

브랜드 단위로 상품을 탐색하고, 좋아요로 취향을 표현하며, 여러 상품을 한 번에 주문하는 감성 이커머스 플랫폼.
유저 행동(좋아요·주문)은 데이터로 쌓여 이후 랭킹·추천 기능의 재료가 된다.

Vol.1(User 도메인) 위에 Brand → Product → Like → Order 도메인을 순차적으로 쌓는 구조이다.

---

## 2. Actors

| Actor | 설명 |
|-------|------|
| 고객 (User) | 상품 탐색, 좋아요, 주문을 수행하는 서비스 이용자 |
| 어드민 (Admin) | 브랜드·상품을 등록·관리하고 주문을 조회하는 운영자 |
| 비인증 사용자 | 회원가입 없이 브랜드·상품을 읽기 전용으로 탐색하는 방문자 |

---

## 3. 서비스 흐름 예시

### 어드민 흐름

1. 어드민이 브랜드를 등록한다
2. 어드민이 해당 브랜드에 상품을 등록한다

### 유저 흐름

1. 유저가 상품 목록을 탐색한다
2. 마음에 드는 상품에 좋아요를 누른다
3. 여러 상품을 한 번에 담아 주문한다
4. 주문 내역을 날짜 범위로 조회한다

---

## 4. 유저 스토리

### Brand

- 고객으로서, 브랜드 정보를 조회해 어떤 브랜드인지 확인하고 싶다
- 어드민으로서, 새로운 브랜드를 등록하고 정보를 수정·삭제할 수 있어야 한다
- 어드민으로서, 등록된 브랜드 목록을 페이지 단위로 조회할 수 있어야 한다

### Product

- 고객으로서, 상품 목록을 탐색하고 브랜드로 필터링하거나 최신순으로 정렬할 수 있어야 한다
- 고객으로서, 특정 상품의 상세 정보(이름, 가격, 재고, 좋아요 수)를 확인할 수 있어야 한다
- 어드민으로서, 상품을 등록·수정·삭제할 수 있어야 하며, 등록 시 유효한 브랜드를 지정해야 한다

### Like

- 고객으로서, 마음에 드는 상품에 좋아요를 등록하고 취소할 수 있어야 한다
- 고객으로서, 내가 좋아요한 상품 목록을 조회할 수 있어야 한다
- 고객으로서, 타인의 좋아요 목록은 조회할 수 없어야 한다

### Order

- 고객으로서, 여러 상품을 한 번에 주문할 수 있어야 한다
- 고객으로서, 재고가 부족한 상품이 하나라도 있으면 주문이 거부되어야 한다
- 고객으로서, 주문 내역을 날짜 범위로 조회할 수 있어야 한다
- 고객으로서, 본인의 주문 상세 내역을 조회할 수 있어야 한다
- 어드민으로서, 전체 주문 목록과 단건 주문 상세를 조회할 수 있어야 한다

---

## 5. 인수 조건

### Brand

- [ ] 어드민은 브랜드를 등록/수정/삭제할 수 있다
- [ ] 브랜드 삭제 시 해당 브랜드의 상품도 함께 삭제된다
- [ ] 고객은 브랜드 ID로 단건 조회할 수 있다
- [ ] 삭제된 브랜드는 조회되지 않는다

### Product

- [ ] 어드민은 상품을 등록/수정/삭제할 수 있다
- [ ] 상품 등록 시 브랜드는 존재하는 브랜드여야 한다
- [ ] 상품 수정 시 브랜드는 변경할 수 없다
- [ ] 고객은 상품 목록을 페이지네이션으로 조회할 수 있다 (브랜드 필터, 최신순 정렬 필수)
- [ ] 삭제된 상품은 목록/상세 조회에 노출되지 않는다

### Like

- [ ] 로그인 유저는 상품에 좋아요를 등록/취소할 수 있다
- [ ] 이미 좋아요한 상품에 중복 등록 시 실패한다
- [ ] 좋아요하지 않은 상품을 취소 시 실패한다
- [ ] 본인의 좋아요 목록만 조회 가능하며, 타인 조회 시 거부된다

### Order

- [ ] 로그인 유저는 여러 상품을 한 번에 주문할 수 있다
- [ ] 동일 상품을 중복 요청 시 실패한다
- [ ] 재고 부족 시 주문 전체가 실패하고 재고는 변경되지 않는다
- [ ] 주문 생성 시 당시 상품명·단가·수량이 기록된다
- [ ] 조회 날짜 범위 미입력 시 실패한다
- [ ] 본인의 주문만 조회 가능하다

## 6. 미결 사항 (이후 단계)

| 항목 | 설명 |
|------|------|
| 결제 (Payment) | 주문 후 결제 흐름 추가 |
| 쿠폰 (Coupon) | 쿠폰 발급·적용 |
| 주문 취소 | 취소 시 재고 복구 포함 |
| 동시 주문 재고 정합성 | 동시성 제어 전략 수립 |
| 랭킹·추천 | 좋아요·주문 데이터 활용 |
