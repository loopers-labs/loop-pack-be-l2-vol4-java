# ADR-023: OrderStatus — PENDING 상태 확정 및 요구사항 문서 정정

- 날짜: 2026-05-29
- 상태: 승인됨

## 결정

주문 생성 시 초기 상태는 **`PENDING`** 으로 확정한다.

`01-requirements.md` section 9에 기술된 아래 내용은 **잘못된 기술**이므로 이 ADR로 대체한다.

> ~~현재는 결제 기능이 없으므로 주문 생성 즉시 `COMPLETED` 단일 상태로 처리한다.~~

## 배경

`01-requirements.md`는 초기 설계 시점에 "결제 없음 → 즉시 COMPLETED"로 작성되었다. 이후 ADR-015에서 다음 이유로 `PENDING`으로 변경되었으나, 요구사항 문서가 함께 업데이트되지 않아 불일치가 발생했다.

## 근거

| 상태 | 의미 |
|---|---|
| `PENDING` | 주문 생성 직후 — 결제 대기 중 |
| `COMPLETED` | 결제까지 완료된 최종 상태 |

결제 게이트웨이 없이 주문 생성 즉시 `COMPLETED`로 전이하면 **실제 비즈니스 상태와 의미가 불일치**한다. 향후 결제 기능(`PENDING → PAID → COMPLETED`) 추가 시 기존 주문 데이터가 이미 `COMPLETED`로 쌓여 있으면 마이그레이션이 필요해진다.

`PENDING`을 유지하면 결제 흐름 확장 시 기존 데이터와 자연스럽게 연결된다.

## ADR-015와의 관계

ADR-015가 이 결정의 원본이며, 이 ADR은 요구사항 문서와의 불일치를 공식적으로 정리하는 목적으로 작성되었다.
