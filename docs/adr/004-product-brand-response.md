# ADR-004: 상품 응답에 브랜드명 포함

- 날짜: 2026-05-20
- 상태: 승인됨

## 결정

상품 조회 응답(`GET /api/v1/products`, `GET /api/v1/products/{productId}`)에 `brandId`와 함께 `brandName`을 포함한다. 상품 조회 시 `BRAND` 테이블을 JOIN하여 브랜드명을 함께 반환한다.

## 근거

요구사항 문서에 "상품, 브랜드 정보 중 고객과 어드민에게 제공되어야 할 정보에 대해 고민해보세요"라고 명시되어 있다. 상품 등록 시 브랜드를 반드시 지정해야 하므로, 브랜드는 상품의 핵심 속성이다.

### 고려한 대안

#### Option 1. brandId만 반환

상품 응답에 `brandId`만 포함하고, 클라이언트가 브랜드명이 필요할 경우 별도로 `GET /api/v1/brands/{brandId}`를 호출하는 방식이다.

```json
{
  "id": 1,
  "brandId": 2,
  "name": "에어맥스",
  "price": 150000
}
```

- **장점**: 상품 응답이 단순해진다. 브랜드명이 필요 없는 상황에서는 불필요한 JOIN이 없다.
- **단점**: 상품 목록을 표시할 때 브랜드명을 함께 보여주려면 클라이언트가 상품 수만큼 브랜드 API를 추가 호출해야 한다(N+1 문제). REST 원칙상 리소스는 자기 서술적(self-descriptive)이어야 하는데, ID만 반환하면 클라이언트가 의미를 파악하기 어렵다.

---

#### Option 2. brandName 포함 응답 (채택)

상품 조회 시 `BRAND` 테이블을 JOIN하여 `brandId`와 `brandName`을 함께 반환하는 방식이다.

```json
{
  "id": 1,
  "brandId": 2,
  "brandName": "Nike",
  "name": "에어맥스",
  "price": 150000
}
```

- **장점**: 클라이언트가 별도 API 호출 없이 브랜드명을 바로 표시할 수 있다. 상품 목록 조회 시 N+1 문제를 방지한다. 응답이 자기 서술적이다.
- **단점**: 상품 조회마다 BRAND JOIN이 추가된다. 브랜드명이 변경될 경우 캐시된 상품 응답과 불일치가 생길 수 있으나, 브랜드명은 자주 변경되지 않으므로 허용 가능하다.
