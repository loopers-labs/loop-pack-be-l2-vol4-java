# ADR-021: 인증 실패 시 401 Unauthorized 반환

- 날짜: 2026-05-29
- 상태: 승인됨

## 결정

인증 인터셉터(`UserAuthInterceptor`)를 통과하는 과정에서 발생하는 모든 실패(헤더 누락, 사용자 없음, 비밀번호 불일치)는 HTTP 401 Unauthorized로 응답한다.

```java
// UserFacade.authenticate()
try {
    UserEntity user = userService.getUser(loginId, password);
    return user.getId();
} catch (CoreException e) {
    throw new CoreException(ErrorType.UNAUTHORIZED, "아이디 또는 비밀번호가 올바르지 않습니다.");
}
```

## 근거

인증 단계에서 발생하는 오류의 원인(사용자 없음 vs. 비밀번호 불일치)을 클라이언트에 구분하여 알려주면, 공격자가 유효한 아이디를 열거(enumeration)하는 데 악용할 수 있다. 따라서 인증 실패 원인을 구분하지 않고 일관되게 401로 응답하는 것이 보안상 바람직하다.

- **헤더 누락**: 인터셉터 수준에서 인증 자체가 불가능 → 401
- **사용자 없음**: 도메인 수준 NOT_FOUND이지만 인증 컨텍스트에서는 401로 변환
- **비밀번호 불일치**: 인증 실패 → 401

### 고려한 대안

#### Option 1. 원인별 다른 HTTP 상태 코드 반환 (기각)

사용자 없음은 404, 비밀번호 불일치는 400, 헤더 누락은 400을 각각 반환하는 방식이다.

- **장점**: 클라이언트가 실패 원인을 세분화하여 처리할 수 있다.
- **단점**: 아이디 존재 여부를 노출하여 계정 열거 공격에 취약해진다.

#### Option 2. 모든 인증 실패를 401로 통일 (채택)

- **장점**: 보안 관점에서 올바른 접근이다. RFC 7235 표준에 부합한다.
- **단점**: 클라이언트 디버깅 시 원인 파악이 어려울 수 있다(내부 로그로 보완).
