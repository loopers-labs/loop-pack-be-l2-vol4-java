# Spec: PRD-3 상품 목록 (admin)

**소스**: `docs/volume-2/01-requirements.md` — PRD-3
**작성일**: 2026-05-27
**상태**: Draft

## 시나리오 요약

관리자가 브랜드 필터·페이지 정보를 선택적으로 지정해 등록된 상품 목록을 조회한다. 관리자 인증을 통과한 요청만 허용하며, 삭제되지 않은 상품 한 페이지 분량을 등록 시각 내림차순으로, 페이지 메타 정보(현재 페이지 번호·페이지 크기·총 페이지 수·총 항목 수)와 함께 반환한다. 각 항목은 식별자·이름·설명·소속 브랜드(식별자·이름)·가격·재고 수량(정확한 정수)·등록 시각·갱신 시각을 포함한다. 대고객 목록(PRD-1)과 달리 재고를 정확한 수량으로 노출하고 좋아요 수는 포함하지 않는다.

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 상품들 존재, **When** page·size로 목록 요청, **Then** 등록 시각 내림차순 항목들과 페이지 메타가 반환된다(200). 각 항목은 식별자·이름·설명·브랜드(식별자·이름)·가격·재고 수량·등록 시각·갱신 시각.
2. **Given** 관리자 인증 + 특정 브랜드 상품 + 다른 브랜드 상품, **When** brandId 필터로 요청, **Then** 해당 브랜드 활성 상품만 반환된다.
3. **Given** 관리자 인증 + 활성 상품 없음, **When** 목록 요청, **Then** 빈 목록과 페이지 메타가 반환된다(200).

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 목록 요청, **Then** 403 FORBIDDEN.
2. **Given** page < 0 또는 size가 허용 범위(1~100) 밖, **When** 목록 요청, **Then** 400 BAD_REQUEST.

### 비즈니스 규칙
- 삭제된 상품은 응답에서 제외.
- 관리자 응답에는 재고 수량을 정확한 정수로 노출(대고객 가용 여부와 대비).
- 페이지 크기 미지정 시 20건, 1~100만 허용.
- 결과는 등록 시각 내림차순 정렬.
- brandId로 지정한 브랜드가 존재하지 않거나 삭제됐으면 빈 목록 (PRD-1과 동일 정책).

## 엣지 케이스
- 빈 결과(빈 목록 + 메타) / size 미지정(20) / size 0·101(400) / page 음수(400) / 마지막 페이지 부분 채움.
- 재고 0 상품도 재고 수량 0으로 노출되고 목록에 포함.
- brandId 미존재·삭제 브랜드 → 빈 목록.

## 기능 요구사항

- **FR-001**: 시스템은 관리자 인증 통과 요청만 허용한다(실패 403).
- **FR-002**: 시스템은 brandId(선택)·page(>=0)·size(1~100, 기본 20)를 수신하고 page·size 위반 시 400으로 거절한다.
- **FR-003**: 시스템은 삭제되지 않은 상품만 등록 시각 내림차순으로 페이징한다.
- **FR-004**: brandId 지정 시 해당 브랜드 활성 상품만 반환하고, 브랜드가 미존재·삭제 상태면 빈 목록을 반환한다.
- **FR-005**: 각 항목에 식별자·이름·설명·브랜드(식별자·이름)·가격·재고 수량(정확한 정수)·등록 시각·갱신 시각을 포함한다.
- **FR-006**: 시스템은 페이지 메타(현재 페이지·크기·총 페이지·총 항목)를 포함한다.

## 관련 엔티티
- **ProductModel**: 읽기(전 필드 + 등록/갱신 시각).
- **BrandModel**: 브랜드 식별자·이름 표시를 위한 조인 대상.
- **신규 토대(plan에서 결정)**: 관리자 상품 목록 projection(식별자·이름·설명·브랜드 식별자·브랜드명·가격·재고 수량·등록/갱신 시각). 좋아요 수·재고 가용 여부는 미포함. PRD-4와 공유. 정렬·필터·페이징 조회 메서드.

## 테스트 계획
| 레벨 | 대상 | 무엇을 단언하는가 |
|------|------|------------------|
| Integration | ProductRepository(신규 조회) | 활성만 / brandId 필터 / 등록 시각 내림차순 / 페이징·총 개수 / 미존재 브랜드 필터→빈 / 브랜드명 조인 |
| E2E | `GET /api-admin/v1/products` | 200 + 메타 키 + 항목 키(식별자·이름·설명·브랜드·가격·재고 수량·등록/갱신 시각) / brandId 필터 / 빈 결과 200 / 403(인증 실패) / 400(size·page 위반) |

## 관련 결정
- 결정 7(soft delete): 삭제 상품 제외.
- admin 인증 403(`/api-admin/**` 인터셉터).
- 페이지네이션 접근(Spring Data + 자체 `PageResponse`)·page/size 검증 위치(Facade 가드)는 BRD-2 패턴을 따르되 **plan에서 확정**.
- 재고 정확 수량 노출(대고객 PRD-1의 가용 여부와 대비) — 같은 ProductModel을 다른 projection으로 매핑.

## 성공 기준 / 범위 밖
- 성공: 위 수용 시나리오·테스트 green. `GET /api-admin/v1/products`가 인증·검증·필터·정렬·페이징·메타를 명세대로 처리.
- 범위 밖: 운영 메모 등 추가 관리자 필드, 등록 시각 외 정렬 옵션, 좋아요 수 노출.
