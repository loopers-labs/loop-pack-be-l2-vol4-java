# Spec: PRD-4 상품 상세 (admin)

**소스**: `docs/volume-2/01-requirements.md` — PRD-4
**작성일**: 2026-05-27
**상태**: Draft

## 시나리오 요약

관리자가 특정 상품의 상세 정보를 조회한다. 관리자 인증을 통과한 요청만 허용하며, 대상 상품이 삭제되지 않고 존재할 때 식별자·이름·설명·소속 브랜드(식별자·이름)·가격·재고 수량(정확한 정수)·등록 시각·갱신 시각을 반환한다. 대고객 상세(PRD-2)와 달리 재고를 정확한 수량으로 노출하고 좋아요 수는 포함하지 않는다.

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 브랜드 소속의 활성 상품, **When** productId로 상세 요청, **Then** 200과 함께 식별자·이름·설명·브랜드(식별자·이름)·가격·재고 수량·등록 시각·갱신 시각이 반환된다.

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 상세 요청, **Then** 403 FORBIDDEN.
2. **Given** 존재하지 않는 productId, **When** 상세 요청, **Then** 404 NOT_FOUND.
3. **Given** 삭제된 상품, **When** 상세 요청, **Then** 404 NOT_FOUND.

### 비즈니스 규칙
- 관리자 응답에는 재고 수량을 정확한 정수로 노출.
- 응답 상품은 삭제되지 않은 상태여야 한다(삭제·미존재 시 404).

## 엣지 케이스
- 재고 0 상품 → 재고 수량 0, 그래도 상세 조회됨.
- 미존재·삭제 상품 → 404.
- 인증 실패가 자원 부재보다 먼저 평가됨(인터셉터 단계 403).

## 기능 요구사항

- **FR-001**: 시스템은 관리자 인증 통과 요청만 허용한다(실패 403).
- **FR-002**: 시스템은 대상 상품이 삭제되지 않고 존재할 때만 응답하며, 미존재·삭제 시 404로 응답한다.
- **FR-003**: 시스템은 식별자·이름·설명·브랜드(식별자·이름)·가격·재고 수량(정확한 정수)·등록 시각·갱신 시각을 반환한다.

## 관련 엔티티
- **ProductModel**: 읽기(전 필드 + 등록/갱신 시각).
- **BrandModel**: 브랜드 식별자·이름 표시를 위한 조인 대상.
- **신규 토대(plan에서 결정)**: 관리자 상품 상세 projection(PRD-3 목록 항목과 동일 필드 집합). 브랜드명 조인 단건 조회.

## 테스트 계획
| 레벨 | 대상 | 무엇을 단언하는가 |
|------|------|------------------|
| Integration | ProductRepository(신규 단건 조회) | 활성 상품 + 브랜드명 조인 / 삭제·미존재 → 부재 |
| E2E | `GET /api-admin/v1/products/{productId}` | 200 + 응답 키(식별자·이름·설명·브랜드·가격·재고 수량·등록/갱신 시각) / 403(인증 실패) / 미존재 404 / 삭제 상품 404 |

## 관련 결정
- 결정 7(soft delete): 삭제 상품 404.
- admin 인증 403(`/api-admin/**` 인터셉터).
- 재고 정확 수량 노출(대고객 PRD-2의 가용 여부와 대비).
- 단건 조회 브랜드명 조인은 PRD-2와 같은 메커니즘을 admin projection으로 매핑 — **plan에서 공유 범위 결정**.

## 성공 기준 / 범위 밖
- 성공: 위 수용 시나리오·테스트 green. `GET /api-admin/v1/products/{productId}`가 인증·활성 판정·브랜드명·정확한 재고를 명세대로 처리.
- 범위 밖: 운영 메모 등 추가 관리자 필드, 좋아요 수 노출.
