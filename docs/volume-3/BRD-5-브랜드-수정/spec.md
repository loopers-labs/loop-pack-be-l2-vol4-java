# Spec: BRD-5 브랜드 수정

**작성일**: 2026-05-25
**상태**: Draft

## 시나리오 요약

관리자가 기존 브랜드의 이름·설명을 수정한다. 관리자 인증을 통과한 요청만 허용하며, 대상 브랜드가 활성 상태로 존재하고, 입력값이 허용 범위(이름 1~50자) 안에 있으며, 새 이름이 자신을 제외한 다른 활성 브랜드와 중복되지 않으면 정보를 갱신하고 식별자를 반환한다.

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 활성 브랜드 존재 + 새 이름이 다른 활성 브랜드와 중복 안 됨, **When** 새 이름·설명으로 수정 요청, **Then** 정보가 갱신되고 식별자가 반환된다(200).
2. **Given** 활성 브랜드 존재, **When** 기존과 동일한 이름으로 수정 요청, **Then** 정상 처리되고 갱신 시각만 새로 찍힌다.

### Exception Flow
1. **Given** 관리자 인증 실패, **When** 수정 요청, **Then** 403 FORBIDDEN.
2. **Given** 대상 브랜드가 부재거나 이미 삭제됨, **When** 수정 요청, **Then** 404 NOT_FOUND.
3. **Given** 관리자 인증 통과, **When** 이름이 빈 값이거나 50자 초과, **Then** 400 BAD_REQUEST.
4. **Given** 새 이름이 자신을 제외한 다른 활성 브랜드와 중복, **When** 수정 요청, **Then** 409 CONFLICT.

### 비즈니스 규칙
- 이름은 1~50자.
- 동일 이름으로의 수정도 정상 처리(갱신 시각만 갱신).
- 중복 검사 대상은 자신을 제외한 활성 브랜드. 삭제된 브랜드 이름은 제외.
- 설명은 선택. 입력값으로 그대로 교체(null 허용).

## 엣지 케이스
- 자기 이름 그대로 수정(통과) / 다른 활성 브랜드 이름(409) / 삭제된 브랜드와 동일 이름(통과).
- 이름 51자(400) / 빈 값(400).
- 대상이 이미 삭제됨(404) / 존재하지 않는 id(404).

## 기능 요구사항
- **FR-001**: 시스템은 관리자 인증 통과 요청만 수정을 허용한다(실패 403).
- **FR-002**: 시스템은 대상 브랜드가 활성 상태로 존재하는지 확인한다(부재/삭제 404).
- **FR-003**: 시스템은 이름이 1~50자인지 검증한다(위반 400).
- **FR-004**: 시스템은 새 이름이 자신을 제외한 활성 브랜드와 중복되지 않는지 검사한다(중복 409).
- **FR-005**: 시스템은 이름·설명을 갱신하고 식별자를 반환한다.

## 관련 엔티티
- **BrandModel**: 자기 속성 갱신 행위(`update(name, description)`) 신규 추가.
- **BrandRepository**: 대상 활성 조회(존재 보장), 자신 제외 활성 이름 중복 검사 메서드 신규.

## 테스트 계획
| 레벨 | 대상 | 무엇을 단언하는가 |
|------|------|------------------|
| Model 단위 | BrandModel.update | 이름·설명 갱신, 이름 위반 시 BAD_REQUEST |
| Service/Facade 단위 | 수정 유스케이스 | 대상 부재 NOT_FOUND, 다른 활성 중복 CONFLICT, 자기 이름 통과, 정상 갱신 |
| Integration | BrandRepository | 자신 제외·삭제 제외 이름 중복 검사 |
| E2E | `PUT /api-admin/v1/brands/{brandId}` | 200 / 403 / 404 / 400 / 409 (statusCode+meta.result+errorCode) |

## 관련 결정
- 결정 7(soft delete): 삭제된 브랜드는 중복 검사·수정 대상에서 제외.
- admin 인증 403, 설명 String 직접 보유(BRD-4 결정 계승).

## 성공 기준 / 범위 밖
- 성공: 위 수용 시나리오·테스트 green. `PUT /api-admin/v1/brands/{brandId}`가 인증·존재·검증·중복·갱신 분기를 명세대로 처리.
- 범위 밖: 동시 수정 race, 브랜드 삭제(BRD-6).
