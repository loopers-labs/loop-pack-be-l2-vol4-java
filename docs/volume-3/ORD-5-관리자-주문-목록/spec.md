# Spec: ORD-5 관리자 주문 목록

**소스**: `docs/volume-2/01-requirements.md` — ORD-5
**작성일**: 2026-05-27
**상태**: Draft

## 시나리오 요약

관리자가 페이지 정보를 담아 전체 회원의 주문 목록을 조회한다. 관리자 인증을 통과한 요청만 허용하며, 전체 회원의 주문 한 페이지 분량을 주문 시각 내림차순으로 페이지 메타와 함께 반환한다. 각 항목에는 주문의 식별자·주문한 회원의 식별자·상태·주문 시각·총 결제 금액이 포함된다(항목 상세는 포함하지 않는 목록 레벨 응답).

## 수용 시나리오 (Given/When/Then)

### Main Flow
1. **Given** 관리자 인증 + 여러 회원의 주문 다수, **When** page·size로 목록을 요청하면, **Then** 전체 주문이 주문 시각 내림차순으로 페이지 메타와 함께 반환된다(200). 각 항목은 주문 식별자·회원 식별자·상태·주문 시각·총 결제 금액.
2. **Given** 관리자 인증 + 주문 없음, **When** 요청하면, **Then** 빈 목록과 페이지 메타가 반환된다(200).

### Exception Flow
1. **Given** 관리자 인증 실패(헤더 누락 또는 `X-Loopers-Ldap` 불일치), **When** 요청하면, **Then** 403 FORBIDDEN.
2. **Given** page < 0 또는 size가 허용 범위(1~100) 밖, **When** 요청하면, **Then** 400 BAD_REQUEST.

### 비즈니스 규칙
- 결과는 주문 시각 내림차순으로 정렬한다.
- 페이지 크기 미지정 시 20건, 1~100만 허용.
- 목록 항목은 주문 헤더 레벨 정보(식별자·회원 식별자·상태·주문 시각·총액)만 포함하고 주문 항목 상세는 포함하지 않는다(상세는 ORD-6).

## 엣지 케이스
- 전체 회원 주문 노출(본인 격리 없음 — 관리자 권한).
- 빈 결과(200 빈 목록 + 메타).
- 페이지: size 미지정(20) / size 0·101(400) / page 음수(400) / 마지막 페이지 부분 채움.
- 권한 경계: 회원 인증 헤더만 있고 admin 헤더 없으면 403.

## 기능 요구사항

- **FR-001**: 시스템은 관리자 인증(`X-Loopers-Ldap: loopers.admin`)을 통과한 요청만 허용해야 한다. 실패 시 403.
- **FR-002**: 시스템은 page(>=0)·size(1~100, 기본 20)를 수신하고 위반 시 400으로 거절해야 한다.
- **FR-003**: 시스템은 전체 회원의 주문을 주문 시각 내림차순으로 페이징해야 한다.
- **FR-004**: 각 항목에 주문 식별자·회원 식별자·상태·주문 시각·총 결제 금액을 포함해야 한다.
- **FR-005**: 시스템은 페이지 메타(현재 페이지·크기·총 페이지·총 항목)를 포함해야 한다.

## 관련 엔티티

- **OrderModel**: 전체 주문(필터 없음) 조회, 주문 시각 내림차순 정렬. 회원 식별자(userId)·상태·총액 노출.
- **재사용**: `AdminAuthInterceptor`(관리자 인증), `OrderRepository`(전체 페이징 조회), `ErrorType`(BAD_REQUEST·FORBIDDEN).
- **조회 형태**: 목록 레벨 정보만 담는 admin projection(항목 미포함)은 plan에서 결정(PRD-3 admin 목록 projection 패턴 참고).

| 레벨 | 대상 | 무엇을 단언하는가 |

## 관련 결정

- **권한**: 관리자 인증은 BRD/PRD admin API와 동일한 `AdminAuthInterceptor`(403). 전체 회원 주문 노출은 관리자 권한 범위.
- **응답 범위**: 목록은 헤더 레벨만, 항목 상세는 ORD-6. (PRD-3 목록 vs PRD-4 상세와 같은 결)

## 성공 기준 / 범위 밖

- **성공**: 위 수용 시나리오·테스트가 green. `GET /api-admin/v1/orders`가 관리자 인증·페이징·정렬·헤더 레벨 응답을 명세대로 처리.
- **범위 밖**: 주문 항목 상세(ORD-6), 회원·기간·상태 필터(본 라운드 미정의), 회원별 그룹핑.
