# 유저 리서치 목적 및 검증 가설

> **한 줄 목적:** 도메인 스팩과 요구사항에서 세운 가설이 실제 IT 기술자에게 유용한지 검증하고,
> 그 결과를 시퀀스 다이어그램 및 전체 흐름 설계에 반영한다.

---

## 왜 지금 리서치를 하는가

코드를 짜기 전에 우리는 이미 많은 것을 가정했다.

- "IT 기술자는 카테고리와 난이도로 도서를 찾는다"
- "출판사 기반으로 탐색한다"
- "좋아요로 관심 도서를 관리한다"
- "단일 주문 1건이 편리하다"

이 가정들이 틀렸다면, 시퀀스를 아무리 정교하게 짜도 쓸모없는 기능이 된다.
**리서치는 시퀀스를 짜기 전 마지막 검증 단계다.**

---

## 검증할 가설 목록

| ID | 가설 | 검증 방법 | 연결된 도메인 개념 |
|----|------|-----------|------------------|
| H1 | IT 기술자는 **기술 카테고리(TechCategory)**로 도서를 필터링한다 | 설문 + 인터뷰 | `TechCategory` enum |
| H2 | IT 기술자는 **난이도(Level)**가 도서 선택에 결정적으로 중요하다 | 설문 + 인터뷰 | `Level` enum |
| H3 | **출판사(Publisher)** 브랜드가 구매 신뢰도에 영향을 준다 | 설문 | `Publisher` 컨텍스트 |
| H4 | **좋아요(Like)** 기능으로 관심 도서를 나중에 다시 찾는다 | 인터뷰 | `Like` 컨텍스트 |
| H5 | 여러 출판사 도서를 **단일 주문 1건**으로 처리하는 것이 편리하다 | 설문 + 인터뷰 | `Order` 컨텍스트 |
| H6 | **가입 시 쿠폰 자동 지급**이 첫 구매를 유도하는 데 효과적이다 | 인터뷰 | `Coupon` 확장 포인트 |

---

## 리서치 결과의 활용

```
설문 결과 분석
    ↓
가설 기각 / 채택 판단
    ↓
인터뷰 인사이트로 보완
    ↓
도메인 스팩 조정 (필요 시)
    ↓
02-sequence-diagrams.md 반영
```

## 페르소나 출처

리서치 대상은 Nemotron-Personas-Korea (NVIDIA) 데이터셋에서 IT 직군을 필터링해 사용한다.

```python
from datasets import load_dataset
ds = load_dataset("nvidia/Nemotron-Personas-Korea")
it_personas = ds.filter(lambda x: x["occupation_category"] == "IT")
# 직군별 샘플: 백엔드·프론트엔드·DevOps·AI·ML·네트워크 엔지니어
```

각 인터뷰 기록의 `페르소나 ID`는 위 데이터셋의 실제 레코드 ID를 참조한다.
