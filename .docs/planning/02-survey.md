# 유저 설문조사

> **목적:** 가설(H1~H6)의 정량적 우선순위를 파악한다.
> **대상:** IT 직군 기술자 (백엔드/프론트/DevOps/AI·ML/네트워크 등)
> **소요 시간:** 3분 이하 / 5문항
> **페르소나 출처:** nvidia/Nemotron-Personas-Korea 데이터셋

---

## 기획자 검토 원칙

```
✓ 기능 이름을 먼저 언급하지 않는다 (confirmation bias 방지)
✓ "~하면 쓸 것 같나요?"가 아닌 "지금 어떻게 하고 있나요?"로 묻는다
✓ 기술 용어보다 일상 언어로 작성한다
✓ 각 문항은 하나의 가설만 검증한다
```

---

## Q1. 기술 도서 탐색 방식 (H1 검증)

**기술서를 찾을 때 주로 어떻게 검색하시나요? 가장 자주 쓰는 방법을 선택해주세요.**

```
○ "Java 책", "쿠버네티스 책"처럼 기술 이름으로 검색한다
○ 출판사 이름으로 검색한다 (예: O'Reilly, 한빛미디어)
○ 커뮤니티/SNS에서 추천받은 제목으로 검색한다
○ 베스트셀러·신간 목록을 훑어본다
○ 기타
```

> **검증:** '기술 이름으로 검색' 비율 → H1(TechCategory) 필터 필요성 판단
> **왜 이렇게 물어보는가:** 기능 이름을 언급하지 않고 자연스러운 행동 패턴을 유도한다.

---

## Q2. 도서 선택 기준 상위 2가지 (H2·H3 복합 검증)

**기술서를 구매할 때 가장 중요하게 보는 것을 2개만 골라주세요.**

```
□ 내 기술 스택과 관련 있는지
□ 내 수준에 맞는 책인지 (너무 쉽거나 어렵지 않은지)
□ 어떤 출판사에서 낸 책인지
□ 저자가 현업 경험이 있는지
□ 출판 연도 (최신 버전 기반인지)
□ 가격 또는 할인 여부
□ 지인·커뮤니티 추천
```

> **검증:** 상위 2개 분포로 H2(난이도)·H3(출판사) 우선순위 결정
> **왜 2개로 제한하는가:** "모두 중요하다" 응답을 막고 실질적 우선순위를 드러낸다.

---

## Q3. 수준 판단의 어려움 (H2 검증)

**기술서를 사기 전에 "내 수준에 맞는 책인지" 확인하기 어려웠던 경험이 있나요?**

```
① 전혀 없다 — 목차나 소개만 봐도 충분하다
② 가끔 있다 — 사고 나서 수준이 안 맞았던 적이 있다
③ 자주 있다 — 수준을 판단하기 어려운 경우가 많다
④ 항상 어렵다 — 수준 표시가 명확했으면 좋겠다
```

> **검증:** ②~④ 응답 비율 → H2(Level enum) 필요성 판단
> **왜 이렇게 물어보는가:** "난이도 표시가 도움이 되나요?"라고 물으면 대부분 "예"라고 답한다.
> 대신 실제 불편 경험을 물어 진짜 필요를 확인한다.

---

## Q4. 관심 도서 관리 방법 (H4 검증)

**"나중에 살 것 같은데 지금은 아닌" 기술서를 어떻게 기억해두시나요?**

```
○ 서점 앱·사이트의 찜/위시리스트 기능을 쓴다
○ 노션, 메모앱 등에 직접 기록한다
○ 브라우저 북마크로 저장한다
○ 따로 기억해두지 않는다
○ 기타
```

> **검증:** '서점 찜 기능' vs '외부 도구 사용' 비율 → H4(Like) 기능의 대체 가능성 판단
> **왜 이렇게 물어보는가:** "좋아요 기능이 있으면 쓰시겠어요?"는 yes-bias를 유발한다.
> 현재 행동을 물어 실제 니즈를 확인한다.

---

## Q5. 복수 도서 구매 경험 (H5 검증)

**기술서를 한 번에 2권 이상 구매한 적이 있나요? 있다면 어땠나요?**

```
○ 자주 있다 — 불편함 없이 잘 쓴다
○ 있었는데 불편했다 — 배송이 따로 오거나 결제가 번거로웠다
○ 없다 — 한 번에 1권씩만 산다
○ 없다 — 한 번에 사고 싶었지만 불편해서 포기했다
```

> **검증:** '불편했다' + '포기했다' 합산 비율 → H5(단일 주문) 설계 타당성 확인
> **왜 이렇게 물어보는가:** "한 번에 결제 vs 따로 결제"라는 개념 질문은 실무 경험 없이 답하기 어렵다.
> 경험 자체를 먼저 묻는다.

## 설문 결과 해석 기준표

| 가설 | 측정 문항 | 채택 기준 | 기각 시 설계 영향 |
|------|---------|---------|----------------|
| H1 카테고리 | Q1 — 기술명 검색 비율 | 50% 이상 | TechCategory 필터 우선순위 하향 |
| H2 난이도 | Q2 상위 2위 이내 + Q3 ②~④ 비율 | Q3 ②~④ ≥ 60% | Level enum 선택 구현으로 격하 |
| H3 출판사 | Q2 — 출판사 선택 비율 | 상위 3위 이내 | Publisher 큐레이션 기능 축소 |
| H4 좋아요 | Q4 — 외부 도구 사용 비율 | '찜 기능' < '외부 도구' 이면 니즈 큼 | Like 기능 설계 강화 또는 재검토 |
| H5 단일 주문 | Q5 — 불편 경험 합산 | ≥ 40% | Order 설계 유지 (불편 해소 포인트) |
