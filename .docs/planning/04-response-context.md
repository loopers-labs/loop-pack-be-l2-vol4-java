# 페르소나 설문 응답 컨텍스트 (Response Context)

> 이 문서는 페르소나 텍스트에서 속성을 추출하여 설문 응답을 생성하는 규칙을 정의한다.
> Python 스크립트는 이 규칙을 그대로 코드로 구현한다. 규칙을 바꾸려면 이 파일을 수정하면 된다.
> LLM 호출 없이 결정론적으로 동작한다.

## 3. 인터뷰 응답 규칙 (20명 샘플)

인터뷰는 페르소나의 **실제 텍스트**에서 키워드를 추출해 템플릿 문장에 채워 넣는다.
LLM 없이 결정론적 템플릿 방식으로 생성한다.

### IQ1. 최근 구매 경험
템플릿: "{occupation} 일을 하면서 {topic} 관련 책이 필요했습니다. {source}에서 찾았고, {method}로 구매했습니다."
- topic: 페르소나 텍스트에서 기술 스택 키워드 첫 번째
- source: Q1 결과에 따라 (A→검색, B→출판사 사이트, C→커뮤니티)
- method: "인터넷 서점"

### IQ3. 수준 불일치 경험
- Q3 결과가 1(없음): "목차를 꼼꼼히 보면 대부분 파악할 수 있습니다."
- Q3 결과가 2(가끔): "가끔 사고 나서 생각보다 기초적인 내용이라 아쉬웠던 경험이 있습니다."
- Q3 결과가 3(자주): "수준 표시가 없어서 사고 나서야 너무 어렵다는 걸 알았던 적이 있습니다."
- Q3 결과가 4(항상): "명확한 난이도 표시가 정말 필요합니다. 맞지 않는 책을 여러 번 샀습니다."

### IQ5. 관심 도서 관리
- Q4 결과 A: "서점 찜 기능을 쓰는데, 여러 서점에 분산되어 있어 한 곳에서 보고 싶습니다."
- Q4 결과 B: "노션에 따로 정리하는데 서점 사이트와 연결이 안 되어 번거롭습니다."
- Q4 결과 C: "브라우저 북마크에 저장하는데 구매 페이지로 바로 안 가는 경우가 있습니다."
- Q4 결과 D: "따로 기억해두지 않다가 잊어버린 적이 꽤 있습니다."

### IQ6. 복수 구매 경험
- Q5 결과 A: "여러 권 사는 게 불편하지 않았습니다. 한 번에 결제하면 됩니다."
- Q5 결과 B: "출판사마다 따로 배송이 와서 언제 다 오는지 추적하기 불편했습니다."
- Q5 결과 C: "보통 한 번에 한 권씩 삽니다. 여러 권 사는 경우가 없었습니다."
- Q5 결과 D: "여러 권 담았는데 결제 과정이 복잡해서 포기하고 한 권만 샀습니다."
