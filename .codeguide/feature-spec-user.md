# 회원 기능 요구사항 명세서

회원가입, 내 정보 조회, 비밀번호 변경 기능에 대한 명세입니다.

---

## 1. 회원가입

### API

```
POST /api/v1/users
```

### 입력 항목

| 항목 | 설명 |
|------|------|
| 아이디 | 영문 대소문자·숫자만 허용, 2~10자 |
| 비밀번호 | 영문·숫자·특수문자 허용 (공백 제외), 8~16자. 단, 생년월일 8자리(예: 19980101)를 포함할 수 없음 |
| 이름 | 한글만 허용, 2~10자 |
| 이메일 | 이메일 형식 준수. `+` `.` `_` `-` 특수문자는 로컬 파트에 허용 |
| 생년월일 | `yyyy-MM-dd` 형식, 오늘 이전의 실제 존재하는 날짜 (예: 2000-02-30 불가) |
| 성별 | `MALE` 또는 `FEMALE` 중 하나. 필수 입력 |

### 정상 응답 (201)

```json
{
  "meta": { "result": "SUCCESS", "errorCode": null, "message": null },
  "data": {
    "id": 1,
    "loginId": "hong123",
    "name": "홍길동",
    "email": "hong@example.com",
    "birthDate": "2000-01-01",
    "gender": "MALE"
  }
}
```

### 오류 응답

| 상황 | 상태 코드 |
|------|----------|
| 입력 항목 형식 위반 | 400 |
| 이미 사용 중인 아이디 | 409 |

## 2. 내 정보 조회

로그인한 회원 본인의 정보를 조회한다.

### API

```
GET /api/v1/users/me
```

인증 헤더(`X-Loopers-LoginId`, `X-Loopers-LoginPw`)를 통해 본인 여부를 확인한다.

### 정상 응답 (200)

```json
{
  "meta": { "result": "SUCCESS", "errorCode": null, "message": null },
  "data": {
    "loginId": "hong123",
    "name": "홍길*",
    "email": "hong@example.com",
    "birthDate": "2000-01-01"
  }
}
```

### 이름 마스킹 규칙

이름의 **마지막 글자 한 자**를 `*`로 가린다. 성별과 내부 식별자(id)는 응답에 포함하지 않는다.

| 원본 | 반환값 |
|------|--------|
| 홍길동 | 홍길* |
| 홍길 | 홍* |

### 오류 응답

| 상황 | 상태 코드 |
|------|----------|
| 존재하지 않는 회원 | 404 |

---

## 3. 비밀번호 변경

로그인한 회원이 새 비밀번호로 변경한다.

### API

```
PATCH /api/v1/users/me/password
```

인증 헤더(`X-Loopers-LoginId`, `X-Loopers-LoginPw`)를 통해 본인 여부를 확인한다.

### 입력 항목

| 항목 | 설명 |
|------|------|
| 새 비밀번호 | 회원가입 비밀번호 규칙과 동일 |

### 변경 조건

- 새 비밀번호가 **현재 비밀번호와 동일하면 변경 불가**

### 정상 응답 (200)

```json
{
  "meta": { "result": "SUCCESS", "errorCode": null, "message": null },
  "data": null
}
```

### 오류 응답

| 상황 | 상태 코드 |
|------|----------|
| 비밀번호 형식 위반 | 400 |
| 현재 비밀번호와 동일 | 409 |
| 존재하지 않는 회원 | 404 |

## 테스트 케이스

### 1. 회원가입

**[단위]** 각 입력 항목의 형식 유효성 검증

```
- 모든 항목이 유효한 경우 → 회원 객체 생성 성공
- 아이디가 1자인 경우 → 실패
- 아이디가 11자인 경우 → 실패
- 아이디에 한글이 포함된 경우 → 실패
- 아이디에 특수문자가 포함된 경우 → 실패
- 비밀번호가 7자인 경우 → 실패
- 비밀번호가 17자인 경우 → 실패
- 비밀번호에 공백이 포함된 경우 → 실패
- 비밀번호에 생년월일 8자리가 포함된 경우 (예: 19980101) → 실패
- 이름이 1자인 경우 → 실패
- 이름이 11자인 경우 → 실패
- 이름에 영문이 포함된 경우 → 실패
- 이름에 숫자가 포함된 경우 → 실패
- 이메일 형식이 올바르지 않은 경우 → 실패
- 생년월일이 미래 날짜인 경우 → 실패
- 생년월일이 존재하지 않는 날짜인 경우 (예: 2000-02-30) → 실패
- 성별을 입력하지 않은 경우 → 실패
```

**[통합]** 실제 DB 저장 및 중복 처리

```
- 유효한 정보로 가입 → DB에 저장되고, 비밀번호는 암호화된 형태로 저장됨
- 이미 사용 중인 아이디로 가입 → 실패 (409)
```

**[E2E]** `POST /api/v1/users`

```
- 모든 항목이 유효한 경우 → 201, 생성된 회원 정보 반환
- 입력 항목 형식 위반 → 400
- 이미 사용 중인 아이디 → 409
```

---

### 2. 내 정보 조회

**[단위]** 이름 마스킹 처리

```
- 세 글자 이름인 경우 (예: 홍길동) → 마지막 글자 마스킹 (홍길*)
- 두 글자 이름인 경우 (예: 홍길) → 마지막 글자 마스킹 (홍*)
```

**[통합]** 회원 조회

```
- 존재하는 아이디로 조회 → 회원 정보 반환
- 존재하지 않는 아이디로 조회 → 반환값 없음
```

**[E2E]** `GET /api/v1/users/me`

```
- 정상 조회 → 200, 마스킹된 이름으로 반환
- 존재하지 않는 회원 → 404
```

---

### 3. 비밀번호 변경

**[단위]** 새 비밀번호 형식 유효성 검증

```
- 회원가입 비밀번호 규칙과 동일한 케이스 적용
  (길이 위반, 공백 포함, 생년월일 포함 등)
```

**[통합]** 변경 조건 및 저장 처리

```
- 유효한 새 비밀번호로 변경 → 새 비밀번호가 암호화되어 저장됨
- 현재 비밀번호와 동일한 값으로 변경 시도 → 실패 (409)
- 존재하지 않는 회원의 비밀번호 변경 시도 → 실패 (404)
```

**[E2E]** `PATCH /api/v1/users/me/password`

```
- 유효한 새 비밀번호로 변경 → 200
- 변경 후 이전 비밀번호로 로그인 시도 → 인증 실패
- 변경 후 새 비밀번호로 로그인 시도 → 200
- 현재 비밀번호와 동일한 값으로 변경 시도 → 409
- 형식에 맞지 않는 새 비밀번호 → 400
```
