# 영속성 지침

JPA Entity, Repository, QueryDSL 기본 규칙은 `.claude/rules/code-conventions.md` 를 따른다.

## 이커머스 영속성 정책

- 논리 삭제가 필요한 도메인 데이터는 논리 삭제 방식으로 처리한다.
- 복잡한 상품 목록 조회, 관리자 검색, 동적 필터는 QueryDSL 사용을 고려한다.

## 주문 상품 스냅샷

- 주문 상품에는 주문 당시 상품명, 브랜드명, 판매가, 수량, 대표 이미지 등 주문 이력 보존용 값을 저장한다.
- 주문 이후 상품/브랜드가 수정 또는 삭제되어도 주문 상세는 스냅샷 기준으로 응답한다.
- `OrderItem` 에 `product_id` 만 저장하는 설계는 피한다.

## 제약 조건

- 좋아요는 `member_id`, `product_id` 조합에 유니크 제약을 둔다.
- 상품은 등록된 브랜드만 참조할 수 있다.
- 상품 수정 시 브랜드는 변경할 수 없다.
- 브랜드 삭제 시 해당 브랜드의 상품도 함께 논리 삭제한다.
