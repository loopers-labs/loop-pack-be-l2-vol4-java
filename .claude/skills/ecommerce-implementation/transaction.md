# 트랜잭션 지침

트랜잭션 경계와 조회 전용 트랜잭션 기본 원칙은 `.claude/rules/code-conventions.md` 를 따른다.

## 주문 생성

주문 생성 트랜잭션은 다음 일관성을 보장한다.

1. 주문 요청 상품 검증
2. 상품 판매 가능 상태 검증
3. 재고 확인과 차감
4. 쿠폰 사용 가능 여부 검증
5. 주문 상품 스냅샷 저장
6. 주문 총액, 할인액, 결제 예정 금액 확정
7. 주문 저장

## 결제

- 결제 요청 전 주문이 결제 가능한 상태인지 확인한다.
- 결제 요청 금액과 주문 결제 예정 금액이 일치해야 한다.
- 결제 성공 시 주문 상태를 결제 완료로 변경한다.
- 결제 실패 시 주문 상태, 재고, 쿠폰 복구 정책을 요구사항에 맞춰 처리한다.

## 이벤트

- 핵심 트랜잭션 성공 이후 행동 기록 또는 이벤트 발행을 수행한다.
- 이벤트 발행 실패가 핵심 트랜잭션을 실패시켜야 하는지는 요구사항별로 분리한다.
- 강한 일관성이 필요하면 outbox 패턴을 검토한다.
