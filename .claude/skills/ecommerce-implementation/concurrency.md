# 동시성 지침

## 재고 차감

- 주문 생성 시 재고 확인과 차감은 초과 판매가 발생하지 않도록 보장한다.
- 낙관적 락, 비관적 락, 조건부 update 중 하나를 선택하고 선택 이유를 남긴다.
- 재고는 음수가 될 수 없다.
- 동시 주문 테스트로 초과 판매 방지를 검증한다.

## 좋아요

- 회원과 상품의 좋아요 관계는 유일해야 한다.
- 중복 좋아요 등록 요청은 멱등 성공 또는 도메인 예외 중 하나로 정책을 명확히 한다.
- 좋아요 취소도 존재하지 않는 상태를 멱등 처리할지 결정한다.

## 쿠폰

- 중복 발급 가능 여부를 명확히 한다.
- 중복 발급이 불가하면 회원/쿠폰 조합의 유니크 제약 또는 원자적 발급 처리를 고려한다.
- 쿠폰 사용과 주문 생성의 일관성을 트랜잭션에서 보장한다.

## 결제

- 결제 요청 재시도와 중복 콜백 가능성을 고려한다.
- 결제 성공/실패 상태 전이는 멱등하게 처리한다.
