# 주문/재고/결제 정합성 가이드

주문, 재고, 쿠폰, 결제 요구사항을 도출할 때 정합성 규칙을 빠뜨리지 않도록 점검한다.

## 주문 생성 흐름

1. 회원을 식별한다.
2. 주문 요청의 상품 ID 와 수량을 검증한다.
3. 상품 존재 여부와 판매 가능 상태를 확인한다.
4. 재고를 확인하고 차감한다.
5. 주문 상품 스냅샷을 저장한다.
6. 쿠폰 사용 조건과 할인 금액을 확정한다.
7. 주문 총액과 결제 예정 금액을 확정한다.
8. 결제 가능한 주문 상태로 전환한다.

## 핵심 정책

- 주문 생성 시 상품 재고 확인과 차감이 보장되어야 한다.
- 주문 상품에는 당시 상품 정보가 스냅샷으로 저장되어야 한다.
- 주문 상세는 상품 원본이 아니라 주문 상품 스냅샷 기준으로 응답한다.
- 결제 성공/실패는 주문 상태 전이에 반영되어야 한다.
- 결제 실패 시 재고/쿠폰 복구 정책은 확인 필요 항목으로 분리한다.

## 예외/조건 분기

- 존재하지 않는 상품이면 주문 실패
- 판매 불가 상품이면 주문 실패
- 주문 수량이 유효하지 않으면 주문 실패
- 재고가 부족하면 주문 실패
- 쿠폰 사용 조건을 만족하지 않으면 주문 실패
- 결제 가능 상태가 아닌 주문은 결제 실패

## 후속 동작

- 주문 생성 행동을 기록한다.
- 결제 성공/실패 행동을 기록한다.
- 결제 이후 추천, 랭킹, 통계 확장 가능성을 고려해 이벤트 후보를 남긴다.
