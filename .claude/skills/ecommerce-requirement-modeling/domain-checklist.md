# 도메인 체크리스트

요구사항 원문에서 DDD 기반 설계 산출물을 만들 때 도메인 후보와 유비쿼터스 언어를 점검한다.

## 유비쿼터스 언어

- 용어는 `.docs/design/01-requirements.md` 의 `유비쿼터스 언어` 섹션에 정의한다.
- ERD, 클래스 다이어그램과 연결될 수 있는 용어는 `회원 (Member)`, `주문 (Order)` 처럼 한글명과 영어명을 함께 적는다.
- 비회원, 관리자처럼 영속 모델보다 역할에 가까운 개념은 영어 엔티티명을 강제하지 않는다.

## 핵심 도메인 후보

- `Member`: 회원가입, 내 정보 조회, 비밀번호 변경
- `Brand`: 브랜드 조회/관리
- `Product`: 상품 조회/관리, 판매 상태, 가격
- `ProductStock`: 주문 가능한 상품 수량
- `ProductLike`: 회원과 상품의 좋아요 관계
- `Coupon`: 발급, 사용 가능 여부, 할인 정책
- `Order`: 주문 생성, 주문 목록/상세 조회
- `OrderItem`: 주문에 포함된 개별 상품 라인
- `OrderItemSnapshot`: 주문 당시 상품명, 브랜드명, 가격 등 주문 상품 이력 보존 정보
- `Payment`: 결제 요청과 결제 결과
- `UserActionLog`: 사용자 행동 기록

## 경계 점검

- 사용자 기능과 관리자 기능을 분리한다.
- 도메인 규칙과 API 표현을 구분한다.
- 주문 이력은 상품/브랜드 변경과 무관하게 재현 가능해야 한다.
- 삭제가 필요한 도메인 데이터는 논리 삭제 정책을 기본으로 둔다.
