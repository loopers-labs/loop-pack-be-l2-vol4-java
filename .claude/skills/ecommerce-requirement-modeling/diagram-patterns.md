# 다이어그램 작성 패턴

설계 산출물의 다이어그램은 `.claude/rules/design-docs.md` 의 규칙을 따른다.

## 이커머스 작성 기준

- 시퀀스 다이어그램은 유스케이스 흐름의 기본 흐름, 예외 분기, 후속 동작을 반영한다.
- 시퀀스 다이어그램은 기능 ID 하나당 하나의 Mermaid `sequenceDiagram` 으로 작성한다.
- 클래스 다이어그램은 유비쿼터스 언어와 핵심 도메인 후보를 기준으로 작성한다.
- ERD 는 도메인 모델을 영속성 모델로 옮길 때 필요한 테이블, 관계, 제약, 인덱스를 표현한다.

## ERD 점검 포인트

- 논리 삭제에 필요한 공통 컬럼 또는 상태를 표현한다.
- 주문 상품 스냅샷 컬럼을 누락하지 않는다.
- 좋아요는 회원과 상품의 조합으로 유일해야 한다.
- 자주 필터링/정렬되는 컬럼은 인덱스 후보로 표시한다.
