## 통합 클래스 다이어그램

<!-- 시스템 전체의 정적 구조 한 장. 공통 VO(Money 등)는 namespace 밖, 도메인은 namespace로 격리. -->

```mermaid
classDiagram
    class <공통 VO 이름> {
        +<타입> value
        +create(value)$ <공통 VO 이름>
        +<도메인 메서드>(<매개변수>) <반환타입>
    }

    namespace <도메인 영어 복수형> {
        class <Entity 이름> {
            +Long id
            +<VO 타입> <필드명>
            +Long <다른 Aggregate의 ID 참조>
            +create(<매개변수>)$ <Entity 이름>
            +<도메인 메서드>(<매개변수>)
        }
        class <VO 이름> {
            +<타입> value
            +create(value)$ <VO 이름>
        }
        class <Enum 이름> {
            <<enumeration>>
            <상태값>
        }
    }

    <Entity> *-- <VO> : <호출 메서드들>
    <Entity> "1" *-- "1..*" <종속 Entity> : <create 시그니처>
```

<!-- Static factory 메서드는 +create(...)$ 표기. composition은 *--, dependency는 ..>. -->
<!-- 메시지 라벨은 composition 화살표에 호출 메서드명들. 예: `Product *-- Stock : create, isAvailable, decrease`. -->

## <도메인 한글> (<도메인 영어>)

<!-- (선택) 본 volume 시나리오에 없지만 다른 도메인이 참조하는 경우 짧은 메타 설명 한 문장. -->

### 다이어그램

<!-- 도메인별 작은 다이어그램. 외부 도메인은 <<external>> placeholder. -->

```mermaid
classDiagram
    class <Entity 이름> {
        +Long id
        +<VO 타입> <필드명>
        +create(...)$ <Entity 이름>
        +<도메인 메서드>(...)
    }
    class <VO 이름> {
        +<타입> value
        +create(value)$ <VO 이름>
    }
    class <외부 도메인 이름> {
        <<external>>
    }

    <Entity> *-- <VO> : <호출 메서드들>
    <Entity> ..> <외부 도메인 이름>
```

### 도메인 모델

<!-- 3열 표. 종류는 Entity / Entity (Aggregate Root) / VO / Enum. -->
<!-- 책임에는 자기 메서드 책임 + 응용 계층이 떠맡는 책임을 함께 명시 (예: "중복 검사·cascade 삭제는 응용 계층 책임"). -->

| 객체 | 종류 | 책임 |
| --- | --- | --- |
| `<Entity 이름>` | Entity (Aggregate Root) | <자기 책임 한 문장 + 응용 계층이 가져가는 책임 한 줄> |
| `<VO 이름>` | VO | <검증·계산 책임 한 줄. 예: "1~50자 검증을 `create()` 팩토리에 단일화."> |
| `<Enum 이름>` | Enum | <상태값 목록 + 추후 추가될 자리 한 줄> |
